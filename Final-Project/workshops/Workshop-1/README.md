### User requirements

We have defined 18 **Functional requirements** and 12 **Non-functional requirements** for the definition of our project. The objective of said requirements, is to create a functional parody of the popular game platform Steam, without significantly exceeding the development purposes of the Software Engineering II course. 

### User stories

We defined illustrative User Stories based on the main functionality of the project. We selected the 17 most relevant User Stories, in order to explain the main functions we aim to achieve in the development of our project.
